论文学习

C-Eval

  1. Introduction

    C-EVAL是一套面向中文场景的综合评测，用来评估大模型的知识与推理能力，弥补主流评测主要面向英文、且对中国语境覆盖不足的问题。

    数据集主要设置如下：

    • 题型为四选一单选题

    • 覆盖52个学科，跨人文、社科、理工等多个门类

    • 分为4个难度层级：初中/高中/大学/职业考试

    • 共13,948道题，含dev/valid/test三个划分

  2. Evaluation

Task 4. Retrieval-Augmented Generation

原理学习

RAG

实验步骤

  1. 复制之前所写的chat函数，还包括client端调用api等

  2. 加载embedding模型nlp_gte_sentence-embedding_chinese-small和task3中存储到本地的FAISS知识库

  3. rag的函数

    • 检索(Retrieve)：先检索与用户提问最相似的3个chunk
    • 增强(Augment)：将检索到的分散的chunk拼接到一个长字符串context_text中，作为参考资料
    • 生成(Generate)：将参考资料context_text和提问user_query通过prompt模板组装在一起，构成输入进大模型的prompt，让大模型进行回答
    • 如果没有找到相似的chunk，就返回不知道，不让模型编造

  4. 测试对比，直接问大模型和有RAG后再问大模型，分别输出回答

Task 5. 大语言模型工具增强

实验步骤

  1. 基于Python Flaskapi和Uvicorn包装成Local RESTFUL API

    调用calculate()函数的接口为/calculate

    并且对接收和返回的json格式进行了规范，对输入的算式使用正则表达式进行安全过滤，只提取数字和计算符号

    使用uvicorn启动后台线程服务，端口为8641

    用eval函数能够得到结果，如果给模型输入计算请求，就调用calculate函数，使用eval函数计算出结果

  2. client端定义和tool参数定义

    client中声明了api等

    tool参数声明了一个工具函数 call_calculator(expression: str)，模型在遇到复杂计算时会调用这个工具

  3. api调用，使用/calculate接口，调用calculate()函数，即用这个接口使用计算器，成功会返回200

  4. agent逻辑的主要函数

    messages

    • system：规定“遇到计算必须用工具”
    • user：用户问题

    第一个chat.completions.create(..., tools=tools_schema)，判断是否需要工具

    • 模型如果判断需要工具，会返回first_msg.tool_calls

    如果存在 tool_calls：

    • 取第一个tool call
    • 解析参数：优先 json.loads，失败则使用ast.literal_eval
    • 执行工具：调用/calculate接口
    • 把tool输出的结果以特定格式回填到messages

    第二个chat.completions.create(...)，不需要tool

    • 让模型结合“工具输出”生成最终的答案

结果展示

运行一个简单的算式测试一下，能够调用外部工具

计算出了准确的结果，并且和计算器计算出来的是一样的

再列出一个复杂一些的算式，普通计算器已经计算不出准确结果了，但是通过大模型调用工具使用eval函数仍然可以完成复杂的大数计算，并且能够计算出准确的结果
