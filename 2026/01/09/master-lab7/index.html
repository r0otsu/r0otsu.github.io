  3. 输入第一题，恶意进程就是伪装成了这个进程，输入[kworker/u8:2-events]正确了

Task 7. git使用

主要

基础篇
(1) git commit

  1. 目标是要提交一次，状态为C2；提交第二次，状态为C3。即两次git commit就可以

  2. 结果展示，成功过关

(2) git branch

  1. 对比目标可以发现，要先使用git branch bugFix创建新的分支，再使用git checkout bugFix移到新创建的bugFix分支

  2. 输入后结果展示

(3) git merge

  1. 看一下目标

    C1-C2: 要先用git branch bugFix创建新分支bugFix，再用git checkout bugFix移到bugFix分支，然后使用git commit从C1提交到C2

    C1-C3：用git checkout main移回main，然后使用git commit从C1提交到C3

    C2/C3-C4：由于最终指向的是main，所以用git merge bugFix将C2合并到main即可

    或者看一下文字版的提示

  2. 结果展示

(4) git rebase

  1. 根据目标图，从最后是bugFix分支，知道应该是由bugFix分支变基到main，所以C2应该是由bugFix提交，C3由main提交，然后再变基

    首先需要新建bugFix分支git branch bugFix

    C1-C2 git checkout bugFix切换到bugFix分支之后，再git commit提交

    C1-C3 git checkout main切换到main之后，再git commit提交

    C2/C3-C4 使用git rebase main将bugFix分支变基到main

  2. 用成功实现

高级话题
(1) 分离 HEAD

  1. 查看目标，其实很简单就是分离C4的头结点

  2. git checkout C4即可分离出C4的头结点

(2) 相对引用(^)

  1. 查看目标，需要先分离出C4的头结点git checkout C4，然后再让HEAD向前一步git checkout HEAD^

  2. 查看结果

(3) 相对引用2(~)

  1. 查看目标，需要将三个指向不同的提交节点

    main->C6：先用一个绝对引用，把main指向C6，因为这个是最后一个提交，所以不用相对引用git branch -f main C6

    bugFix->C0：然后bugFix可以用HEAD~2的相对引用，指向HEAD的前两个提交git branch -f bugFix HEAD~2

    HEAD->C1：HEAD可以直接指向C1git checkout C1

  2. 查看结果

    而pushed是远程分支，用git revert HEAD重新提交上一次提交的内容

    首先需要使用git rebase -i HEAD~2来交换C2 C3的提交顺序，即通过交互的方式调整当前caption分支所指向的前两次提交

  5. 然后main指向C2’‘‘即可，使用git branch -f main C3''强制main指向C3’’，成功通关

远程

Push & Pull —— Git 远程仓库
(1) git clone

  1. git clone会新建一个本地仓库，而不是单纯地创建一个分支

  2. 输入命令后成功过关

(2) 远程分支

  1. 查看目标

    先在main上提交一次git commit，本地仓库的本地分支和远程分支都会得到C3

    然后切换到本地的远程分支git checkout o/main

    再做一次提交git commit，git就能得到分离 HEAD 的状态

    添加新的提交时o/main也不会更新，因为o/main只有在远程仓库中相应的分支更新了以后才会更新

  2. 查看结果，成功通关

(3) git fetch

  1. 查看目标

    git fetch即可，将本地仓库中的远程分支更新成了远程仓库相应分支最新的状态，但是git fetch只是将分支下载，不会更新本地文件

  2. 查看结果

(4) git pull

  1. git pull其实就是在本地仓库的本地main分支中执行git fetch+git merge o/main

    查看目标，git pull即可

  2. 查看结果

(5) git push

  1. git push会把本地仓库的提交上传到远程仓库，并在远程仓库上合并新的提交记录

    查看目标，先执行两次git commit，得到C2 C3之后

    执行git push，远程仓库会更新C2 C3的提交，并且本地和远程仓库都会更新o/main，指向最新的提交

  2. 查看结果
